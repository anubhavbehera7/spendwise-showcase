Architecture Overview

SpendWise is built using MVVM architecture to keep user interface elements separate from application logic. ViewModels handle data processing and state management while repositories interact with Firebase for authentication, expense data, and budgets.

Design Patterns

The project includes design pattern use such as a Factory pattern for creating chart components and a Strategy pattern for switching between sorting behaviors in expense lists.

Architecture Layers

  • UI Layer: Activities and Fragments that display the user interface and handle user interactions
  • ViewModels: Handle data processing and state management, keeping UI separate from business logic
  • Repositories: Interact with Firebase for authentication, expense data, and budgets
  • Data Storage: Firebase backend for real-time data synchronization and cloud storage
